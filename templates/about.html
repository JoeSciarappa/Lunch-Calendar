Name

SMB Linux Break Manager

Description

SMB Linux Break Manager is a tool to schedule and administer break times.

Term Definitions

Name -- Your name. Since there is no authentication method the application will allow you to select any name from the list.
Start Time -- This is the time you initate your break
End Time -- This is the time your break ends.
Mode -- Select an option that best defines your work focus for the day. Options: Phones, Chats, Alerts, Tickets, Other
Action -- add or remove break defined by Name, Start Time, End Time, and Mode

Application Behavior Notes

  • You may add more than one break
  • There is no warning or limit on the amount of time you can schedule a break
  • A warning will display in the header of the homepage if more than two Rackers of the same mode have scheduled a break that overlaps
  • If 'auto' is in either start time or end time, it will try to place you into the least dense interval. Example) If you selected a start time of 11:00 AM but have 'auto' as the end time, it will give you a 1 hour break slot without considering your entered start time
  • If adding more than one break, mode must be the same for all breaks
  • Composit break schedule will show all submitted breaks dispite selected mode. The application will spawn a new, mode specific, graph for each mode that has a population greater than zero

Usage

Add Break - pick name, start time, end time, and mode per your preference. Set action to 'Add' then press submit
  • Start time must be before end time
  • New breaks must not overlap with existing breaks
Remove Break - pick the same name, start time, end time, and mode of the break you wish to remove. Set action to 'Remove' then press submit
  • If auto is selected for start time or end time the application will remove all occurances for that name
Submitting a break with curl
  • You may send a POST request using curl to schedule a break. The payload will need to have values for name, start, end, mode and action
    • name -- format: first name + %20 + last initial
    • start -- format: hh:m0:00
    • end -- format: hh:m0:00
    • mode -- options: phones, chats, tickets, alerts, other
    • action -- options: add , remove
  • Example
    • name="Joe%20S";start="12:00:00";end="13:00:00";mode="phones";action="add"; curl --data "Name=$name&Start%20Time=$start&End%20Time=$end&Mode=$mode&Action=$action" https://lunch.josephsciarappa.com 2>&1 | grep -A 2 "<div id='message'>" | tail -n1
